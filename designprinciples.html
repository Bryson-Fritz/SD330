Bryson's Principles

Visibility

  • Design and format needs to be appealing and structured well
  • Use appropriate UI for the system and your users
  • Provide feedback for user
  • Use smart color selection, like do not write navy blue words on a black background, that is hard for people to see who aren't even color blind
  • Ensure font is big enough or just adjustable so it is suitable for the user
  • Provide error messages if needed
  • Use icons in your design but make sure they are obvious or globally familiar
  • Avoid clutter, it is unorganized, may distract user, and it gets confusing

    • Consistency

  • Use similar functionality and symbols/icons as other applications so user doesn't get confused (things like home page in top left, dropdown menus, hamburger icon, search bar at top middle/right, etc.)
  • Abstrain from implementing potentially confusing updates for users
  • Maintain stability within your system
  • Maintain a standard terminology within the system

    • Feedback

  • Provide users with quick feedback (within .1 seconds)
  • Use visual, auditory feedback mechanisms to correspond to the users needs
  • Apply to use of feedback but allow user to turn it off if they desire to
  • Use feedback because it gives clear communication to the user

    • Constraints

  • Define constraints to limit the users ability within the system
  • Prevent user from performing some action on system
  • Use constraints in a system because we don't want user to do something or because they haven't done anything that would allow them to do some action
  • Leverage constraints to enhance safety within a system, we do not want a user to do something they weren't/aren't supposed to do

    • Affordances (Hint)

  • Ensure user knows what elements of system are interactive, can be clicked, tampered with
  • Use visual clues to let user know about something that should be done
  • Allow user to learn on their own, but help them out a bit

    • Accessibility

  • Ensure system can be used by all users, that includes those with disabilities
  • Provide alternative methods of interactions, some people may be able to do certain things, remember things, see things, etc.
  • Comply with the needs for a variety of users
  • Consider disabilities like color blindness, cognitive, physical, auditory, eyesight

    • Emotional

  • Consider emotions when it comes to the interaction with a system
  • Address potential sources of pleasure vs frustration in the system
  • Understand what emotions the user may be feeling
  • Consider using feedback tools to gain data and "feedback" from user on how the system makes them feel
  • Design elements that will please user, important for UI, UX, HCI
  • Avoid clutter and inaccurate feedback for it may anger the user

    • Cognitive

  • Ackowledge the cognitive load you may be putting on your users, you don't want to give them too much information
  • Aim to minimize mental effort within a system, have the user practically not think and just do
  • Provide a straightforward experience, maybe use breadcrumbs, direct links, search bars

    • Complexity

  • Strive for simplicity in user interfaces
  • Allow some complexity is a system
  • Allow users to learn by doing
  • Use functionality in your interfaces, maybe have some be different from other applications

    • Conceptional

  • Clarify models in a system, make it intuitive for users to understand
  • Provide clear guidance to user throughout their experience
  • Utilize metaphors, isomorphic, skeuomorphic (depending on users), in your design

    • Gender/Cultural

  • Consider cultural and gender related factors when you're making your design
  • Ensure the system is not offensive to any genders or culturals
  • Avoid stereotypes in your design
  • Acknowledge that symbols/signs mean different things amongst different cultures and countries

    • Errors

  • Provide informative error messages
  • Prevent data loss when errors occur
  • Implement guides for different error messages
  • Display errors in a different color than the rest of the page, usually red is appropriate
